## 二、入场信号设计规范

### 2.1 主信号（必选一个）
- 趋势交叉：均线交叉、指标交叉（如 VAR/OTT）
- 突破信号：价格突破关键位
- 形态信号：K线形态识别

### 2.2 确认过滤器（至少选 2 个）
| 过滤器 | 指标 | 目的 |
|--------|------|------|
| 趋势强度 | ADX > 25 | 避免震荡市 |
| 趋势方向 | +DI > -DI（做多）/ 反之 | 方向确认 |
| 成交量 | volume > SMA(volume, 20) | 避免冷清时段 |
| RSI | 35-65（做多）/ 35-65（做空 | 避免极值区 |
| 布林带 | 不在上轨外做多 / 不在下轨外做空 | 避免追涨杀跌 |

### 2.3 多时间框架确认（强烈推荐）
```python
# 必须实现 informative_pairs()
def informative_pairs(self):
    pairs = self.dp.current_whitelist()
    return [(pair, '4h') for pair in pairs]  # 根据策略时间框架选择

# 4h 趋势作为过滤器
# 做多: 4h close > 4h EMA50
# 做空: 4h close < 4h EMA50
```

### 2.4 入场铁律
- **永远不要**只用一个条件入场
- **永远不要**在趋势相反方向入场（多时间框架冲突时不开仓）
- 入场条件用 `reduce(lambda x, y: x & y, conditions)` 确保所有条件满足

## 三、出场信号设计规范

### 3.1 出场层次（按优先级）
1. **止损出场** — 保护本金（最高优先级）
2. **趋势反转出场** — 持仓逻辑不再成立
3. **止盈出场** — ROI / trailing_stop
4. **时间止损** — 超时未盈利退出
5. **极端信号出场** — 过热/过冷信号（如 ADX > 60）

### 3.2 止损规范

#### 静态止损（fallback）
- 合约：`stoploss = -0.50` 到 `-0.80`（宽止损作为安全网）
- 现货：`stoploss = -0.15` 到 `-0.25`

#### ATR 动态止损（推荐 custom_stoploss）
```python
use_custom_stoploss = True

def custom_stoploss(self, pair, trade, current_time, current_rate, current_profit, **kwargs):
    # ATR 止损
    atr_sl = -(atr * multiplier) / current_rate

    # 移动止盈（盈利超过阈值后启用）
    if current_profit > trailing_threshold:
        trailing_sl = current_profit - 0.02
        return max(atr_sl, trailing_sl)

    # 时间止损（超时未盈利收紧）
    if hold_hours > 24 and current_profit < 0:
        return max(atr_sl, -0.05)

    return atr_sl
```

#### trailing_stop 止损
```python
trailing_stop = True
trailing_stop_positive = 0.15     # 盈利15%后启动移动止损
trailing_stop_positive_offset = 0.30  # 需要盈利30%后才开始移动
trailing_only_offset_is_reached = False  # False = 到达offset后按positive距离追踪
```

### 3.3 止损关键规则
- **`use_custom_stoploss = True` 会覆盖 `trailing_stop`** — 二者只能选一
- 如果策略核心依赖 trailing_stop（如 OTT 策略），不要用 custom_stoploss
- custom_stoploss 返回值是 **比率**（如 -0.05 = -5%），不是价格

### 3.4 出场信号设计
```python
def populate_exit_trend(self, dataframe, metadata):
    # 1. 趋势反转（最重要）
    dataframe.loc[trend_reversal_condition, 'exit_long'] = 1

    # 2. 趋势衰竭（ADX 从高位回落）
    dataframe.loc[
        (dataframe['adx'].shift(1) > 60) & (dataframe['adx'] < dataframe['adx'].shift(1)),
        'exit_long'
    ] = 1

    # 3. 极端信号
    dataframe.loc[dataframe['rsi'] > 85, 'exit_long'] = 1
```

### 3.5 出场铁律
- 做多和做空必须有不同的出场条件
- 出场条件不能太敏感（会截断趋势利润）
- 趋势跟踪策略应让 trailing_stop 处理后80%的出场
- **不要**在强趋势中使用 ADX > 60 同时退出多空双方（ADX 高是趋势强，不是反转信号）

## 四、风险管理规范

### 4.1 单笔风险
- 单笔最大亏损 ≤ 账户 2%
- 杠杆越高，仓位越小。公式：`stake = base_stake * (3 / leverage)`
- 永远不要让单笔亏损超过总资金的 5%

### 4.2 总敞口
- 最多同时持仓 ≤ 6 个
- 总保证金占用 ≤ 账户 30%
- 同一板块（如 Layer1/Meme/DeFi）最多 2 个

### 4.3 日风控熔断
```python
# 日亏损 > 20% → 停止交易
# 连续亏损 > 3 笔 → 冷却 2 小时
# 单日交易 > 10 笔 → 暂停检查
```

### 4.4 Protection 配置（config.json）

## 五、杠杆管理规范

### 5.1 杠杆铁律
- **杠杆必须动态调整**，不能固定一个值
- 高波动 = 低杠杆，低波动 = 可用高杠杆
- **ATR/Price > 5% → 强制降至最低杠杆**（波动率熔断）
- 最大杠杆不超过 30x（教程强烈建议上限 25x）

### 5.2 杠杆分级模型
```python
def leverage(self, pair, current_time, current_rate, proposed_leverage, max_leverage, ...):
    # 波动率熔断（第一优先级）
    if atr_ratio > 0.05:
        return min(base_leverage, max_leverage)

    # 趋势强度分级
    if adx > 40 and strong_trend:
        return min(max_leverage, 25)    # 强趋势: 高杠杆
    elif adx > 35:
        return min(max_leverage * 0.75, 18)
    elif adx > 30:
        return min(max_leverage * 0.5, 12)
    elif adx > 25:
        return min(base_leverage * 2, 7)
    else:
        return base_leverage               # 弱趋势: 基础杠杆

    # ADX < 18: 进一步降低杠杆
```

### 5.3 清算保护
```python
def custom_exit(self, pair, trade, current_time, current_rate, current_profit, **kwargs):
    # 接近清算价时强制退出（85% of liquidation distance）
    liq_warning = -(1.0 / leverage) * 0.85
    if current_profit < liq_warning:
        return 'liquidation_risk'
    return None
```

## 六、仓位管理规范

### 6.1 动态仓位模型
```python
def custom_stake_amount(self, pair, current_time, current_rate,
                        proposed_stake, min_stake, max_stake, ...):
    # 1. 波动率调整
    if atr_ratio > 0.04:
        stake = proposed_stake * 0.4    # 高波动减仓60%
    elif atr_ratio > 0.025:
        stake = proposed_stake * 0.7    # 中波动减仓30%

    # 2. 趋势强度调整
    if adx > 35:
        stake = stake * 1.2             # 强趋势加仓20%

    # 3. 杠杆调整（高杠杆=小仓位）
    stake = stake * (3.0 / leverage)

    return max(min_stake, min(stake, max_stake))
```

### 6.2 仓位铁律
- 始终返回 `max(min_stake, min(stake, max_stake))`
- 波动率上升 → 仓位下降（反比关系）
- 杠杆上升 → 仓位下降（反比关系）
- 永不在亏损后加仓（马丁格尔策略禁止）

## 七、Hyperopt 超参优化规范

### 7.1 参数空间定义
```python
from freqtrade.strategy import IntParameter, DecimalParameter, CategoricalParameter

# 只优化影响最大的参数，总数控制在 10 个以内
buy_adx_threshold = IntParameter(20, 45, default=25, space='buy')
buy_volume_factor = DecimalParameter(0.5, 2.0, default=1.0, space='buy')
atr_multiplier = DecimalParameter(1.5, 4.0, default=2.5, space='stoploss')
```

### 7.2 优化执行
```bash
freqtrade hyperopt \
  --strategy <StrategyName> \
  --hyperopt-loss SharpeHyperOptLoss \  # 合约首选
  --spaces buy sell stoploss \          # 分批优化，不要一次优化所有
  --epochs 500 \
  --timerange 20240101-20250601 \       # 至少6个月数据
  -j 4
```

### 7.3 Loss 函数选择
| Loss 函数 | 适用场景 |
|-----------|---------|
| `SharpeHyperOptLoss` | **合约策略首选**，惩罚波动率 |
| `SortinoHyperOptLoss` | 只关心下行风险 |
| `MaxDrawDownHyperOptLoss` | 本金保护优先 |
| `OnlyProfitHyperOptLoss` | 容易过拟合，不推荐 |

### 7.4 过拟合预防
- 训练集 ≥ 6 个月
- 每次优化参数 ≤ 10 个
- epochs ≤ 500
- **必须**用最后 25% 数据做样本外验证
- 训练集/测试集收益差距 > 30% → 过拟合，重新调整

## 八、策略评估标准

### 8.1 评分权重
```
总分 = 收益分*0.40 + 风险分*0.30 + 稳定分*0.20 + 实用分*0.10
```

### 8.2 等级标准
| 等级 | 分数 | 状态 |
|------|------|------|
| A | 8.0+ | 可实盘（保守仓位） |
| B | 7.0-8.0 | 继续优化，可小资金试 |
| C | < 7.0 | 不可实盘，需要重设计 |

### 8.3 最低实盘标准
- 回测区间 ≥ 6 个月
- 交易次数 ≥ 30
- 胜率 ≥ 50%
- 最大回撤 ≤ 25%
- 利润因子 ≥ 1.2
- 样本外验证通过
- Dry-run ≥ 2 周且与回测偏差 ≤ 50%

## 十、常见错误（禁止事项）

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| custom_stoploss 覆盖 trailing_stop | 胜率暴跌 | 二选一，不可同时启用 |
| 杠杆 30x + stoploss -0.80 | 止损触发前已清算 | 杠杆越高止损越紧，或设清算保护 |
| 入场条件单一 | 大量假信号 | 至少3个确认条件 |
| 多空出场条件相同 | 逻辑矛盾 | 做多做空独立写 |
| 出场条件太敏感 | 截断趋势利润 | 让 trailing_stop 处理主要出场 |
| 直接修改实盘策略不测试 | 可能大幅亏损 | 必须：回测 → Dry-run 2周 → 实盘 |
| 多 bot 共用数据库 | 数据混淆 | 每个 bot 独立 db_url |
| kill -9 强杀 bot | 数据库锁损坏 | SIGTERM + 等5秒 |
| API Key 开通提现 | 安全风险 | 仅交易权限 |
| Cross Margin + 高杠杆 | 连环爆仓 | 必须 Isolated |

## 十一、策略持续优化循环

```
回测 → 分析 → 改进 → 回测验证 → Dry-run 2周 → 实盘小资金 → 监控 → 优化
  ↑                                                                    |
  └────────────────────────────────────────────────────────────────────┘
```

- 每次只改一个维度（入场/出场/风控/杠杆）
- 改后必须回测 + 样本外验证
- 优化效果记录到 `optimization_logs/`
- 实盘表现与回测偏差 > 50% → 暂停检查原因
