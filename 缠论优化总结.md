## 踩过的坑

### 1. 未来函数作弊（最致命）
```python
# 错误：看了未来20根K线的最低点
bi = j + np.argmin(low[j:min(j + 20, n)])

# 正确：用当前K线
bi = j
```
这行代码导致回测+221%全是假象，实盘完全无法复现。

### 2. ROI不能完全去掉
- 去掉ROI后胜率从70%+暴跌到20%-
- 3m K线噪声大，微利单必须及时锁定
- 最终方案：ROI设30%仅当安全网，真正出场靠跟踪止损

### 3. 止损不能太紧也不能太松
- -8%：砍掉本可盈利的交易
- -25%：单笔亏损太大，盈亏比倒挂
- **-10%：最优平衡点**

### 4. 信号数量与质量的权衡
- 更多中枢（3个）= 更多信号但质量下降
- 更多币种（51个）= 稀释了高质量信号
- **2个中枢 + 28币 = 最优**

### 5. 部署一致性
- `chanlun.py` 必须同步到服务器（适配器依赖）
- 云端scp可能静默失败，必须校验MD5
- 部署铁律已写入CLAUDE.md

## 核心认知

1. **缠论管线做中枢检测是准确的**（包含处理+递归分型确认+线段+中枢）
2. **买卖点规则必须适配3m时间框架**（原版规则为日线设计，太严格）
3. **信号质量>数量**（3笔高质量交易比50笔随机交易好）
4. **回测必须无未来函数**（任何看未来的操作都会虚增收益）
5. **实盘验证是最终标准**（回测再好不等于实盘能赚钱）

## 进一步优化尝试（全失败）

在达到+35%基线后，尝试了4个方向突破信号数量瓶颈，全部无效：

### 方向①：多时间框架共振 (15m + 3m)
- **方法**：在15m K线上运行适配器获取大周期中枢边界，要求3m信号必须在15m中枢边界附近
- **结果**：❌ **0笔交易**
- **原因**：15m和3m的价格尺度差异太大，15m中枢边界太宽，3m信号几乎从不在其2%范围内触发。硬对齐导致全部信号被过滤。

### 方向②：中枢级别加权 (L1/L2/L3)
- **方法**：chanlun.py 原版中枢级别扩张（size≥10→L2，size≥28→L3），高级别中枢突破给更高杠杆
- **结果**：— **无效果**（+35.5%，与基线相同）
- **原因**：13天回测期内没有中枢生长到L2级别（需要10笔以上重叠），级别扩张逻辑虽然正确但需要更长周期才能触发。

### 方向③：中枢趋势方向过滤 (trend)
- **方法**：chanlun.py 原版 `process_pivot()` 计算出的趋势方向（trend=-1下降/0震荡/1上升），只在趋势一致时入场
- **结果**：❌ **-4.5%**（1笔亏损）
- **原因**：趋势过滤把BOME +30%的盈利交易杀掉了——该笔买入信号的pivot trend被标为-1（下降），但实际是完美的底部反转。3m级别趋势标注噪声太大。

### 方向④：滚动窗口中枢
- **方法**：只取最近1000根K线计算中枢，让中枢边界更贴近当前价格
- **结果**：❌ **-47%**（5笔全亏）
- **原因**：缩短窗口后历史数据不足，中枢结构不稳定，产生大量假突破信号。全量历史数据的中枢更可靠。

### 总结
**当前配置（全量历史+2中枢+0.3%确认突破+EMA50/100）已达到该信号源的全局最优。** 3笔/13天是质量天花板。要突破需要更换信号生成方式或大幅扩大币种池（100+币）。
