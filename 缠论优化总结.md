## 踩过的坑

### 1. 未来函数作弊（最致命）
```python
# 错误：看了未来20根K线的最低点
bi = j + np.argmin(low[j:min(j + 20, n)])

# 正确：用当前K线
bi = j
```
这行代码导致回测+221%全是假象，实盘完全无法复现。

### 2. ROI不能完全去掉
- 去掉ROI后胜率从70%+暴跌到20%-
- 3m K线噪声大，微利单必须及时锁定
- 最终方案：ROI设30%仅当安全网，真正出场靠跟踪止损

### 3. 止损不能太紧也不能太松
- -8%：砍掉本可盈利的交易
- -25%：单笔亏损太大，盈亏比倒挂
- **-10%：最优平衡点**

### 4. 信号数量与质量的权衡
- 更多中枢（3个）= 更多信号但质量下降
- 更多币种（51个）= 稀释了高质量信号
- **2个中枢 + 28币 = 最优**

### 5. 部署一致性
- `chanlun.py` 必须同步到服务器（适配器依赖）
- 云端scp可能静默失败，必须校验MD5
- 部署铁律已写入CLAUDE.md

## 核心认知

1. **缠论管线做中枢检测是准确的**（包含处理+递归分型确认+线段+中枢）
2. **买卖点规则必须适配3m时间框架**（原版规则为日线设计，太严格）
3. **信号质量>数量**（3笔高质量交易比50笔随机交易好）
4. **回测必须无未来函数**（任何看未来的操作都会虚增收益）
5. **实盘验证是最终标准**（回测再好不等于实盘能赚钱）
